# FileServer — 8-Feature Redesign Spec

**Date**: 2026-06-14
**Status**: Proposed

## Overview

Eight features across three layers: UI layout, file viewing, and real-time communication.

| # | Feature | Approach |
|---|---------|----------|
| 1 | Two-panel resize | Draggable `<ResizeHandle>` between sidebar and preview |
| 2 | HTML preview | Sandboxed `<iframe>` with "View Source" toggle |
| 3 | MD outline | TOC sidebar extracted from headings, IntersectionObserver |
| 4 | Light pastel theme | CSS custom properties, `data-theme` toggle, light/dark |
| 5 | Multi-tab preview | Tab bar, max 10 tabs, independently closable |
| 6 | WebSocket push | `@fastify/websocket`, server broadcasts upload events |
| 7 | Tree view | Lazy-load expandable tree replaces flat file list |
| 8 | Context menu | Right-click on tree nodes for file operations |

## Feature Designs

### Feature 1: Two-Panel Resizable Divider

**Component**: `ResizeHandle`

```
Props: onResize(delta: number) => void
```

- 4px wide vertical bar between sidebar and main panel
- CSS cursor: `col-resize`
- On `mousedown`: start tracking, add `mousemove` + `mouseup` listeners on `document`
- On `mousemove`: compute delta from initial position, clamp sidebar width to `[180px, 500px]`
- On `mouseup`: remove listeners
- Sidebar width stored as CSS variable `--sidebar-width` on `.app` element, updated via `style.setProperty`
- Visual states: idle (4px transparent), hover (2px accent glow), active/dragging (2px solid accent, wider hit area via pseudo-element)

**CSS**: `ResizeHandle` rendered in `.app-body` between sidebar and main panel.

---

### Feature 2: HTML Preview (iframe + Source Toggle)

**Component**: `HtmlViewer`

```
Props: { url: string; filePath: string }
```

**States**: loading, rendered, source, error

- Fetch HTML content via `fetch(url).then(r => r.text())`
- **Rendered view** (default): `<iframe sandbox="allow-scripts" srcDoc={htmlContent} />` fills the preview area. Sandbox prevents navigation and top-level script access.
- **Source view**: Toggle button ("View Source" / "View Rendered") switches to raw HTML in `<pre>` block (reuses TextViewer styling).
- Error state: fetch failure, empty content, or file too large (>10MB shows a warning suggesting download instead).

**FileType mapping**: `getFileType()` returns `'html'` for `.html`/`.htm` extensions (currently returns `'text'`).

---

### Feature 3: Markdown Outline / TOC

**Modification**: `MarkdownViewer`

- After markdown content is fetched, extract headings using regex: `/^(#{1,3})\s+(.+)$/gm`
- Each heading: `{ id, level: 1|2|3, text }`. IDs generated by slugifying text (lowercase, replace spaces with dashes, remove special chars).
- TOC rendered as `<nav>` sidebar (200px, collapsible). Shows nested headings with indentation.
- Clicking a TOC entry scrolls the heading into view (`document.getElementById(id).scrollIntoView()`).
- **Active heading tracking**: `IntersectionObserver` on heading elements. The currently visible heading gets highlighted in the TOC.
- ReactMarkdown heading components modified to include `id` attributes.
- On narrow screens (<1024px): TOC collapses to a floating button, opens as overlay.

**Layout**: MarkdownViewer becomes a flex row: `[TOC sidebar] [content]`.

### Feature 4: Light Pastel Theme

**Implementation**: CSS custom properties with `data-theme` attribute

```css
[data-theme="light"] {
  --color-bg: #faf9f7;           /* warm white */
  --color-surface: #f5f4f1;      /* off-white card */
  --color-surface-hover: #eeedf0; /* lavender-tinted hover */
  --color-border: #e2dfe0;       /* muted gray-lavender */
  --color-text: #3b3838;         /* soft charcoal */
  --color-text-muted: #8b8690;   /* muted purple-gray */
  --color-accent: #87a980;       /* sage green */
  --color-accent-hover: #6d8f66; /* deeper sage */
  --color-danger: #c46666;       /* muted red */
  --color-success: #6ba87a;      /* muted green */
}
```

Pastel accent variations for different UI elements:
- **Green** (`#87a980`): primary accent, buttons, active states
- **Purple** (`#b8a5c9`): folder icons, directory highlights
- **Pink** (`#d4a5a5`): delete/danger actions
- **Blue** (`#8aafc8`): info, links, selection

**ThemeToggle component**: Sun/moon icon button in header. Reads/writes `localStorage.setItem('theme', 'light'|'dark')`. Sets `document.documentElement.dataset.theme`. Default: `'dark'`.

**Transition**: `transition: background-color 0.3s, color 0.3s` on `body` for smooth theme switching.

### Feature 5: Multi-Tab Preview

**State in App.tsx**:
```typescript
interface Tab {
  id: string;        // unique id (crypto.randomUUID or counter)
  file: FileItem;
  currentPath: string;
}

const [tabs, setTabs] = useState<Tab[]>([]);
const [activeTabId, setActiveTabId] = useState<string | null>(null);
```

**Behavior**:
- Clicking a file in FileTree: if a tab for that file+path already exists, activate it. Otherwise, create a new tab and activate it.
- Max 10 tabs. Opening an 11th removes the least-recently-active tab (not the current one).
- Closing a tab: if it's the active tab, activate the nearest tab (prefer right neighbor, then left). If it's the last tab, show empty state.
- Tabs survive directory navigation (tabs are independent of current tree path).

**TabBar component**:
- Horizontal bar between main-panel-header and content area.
- Each tab: `[file icon] [filename] [× close]`. Active tab has accent underline.
- Scrollable horizontally if tabs overflow.
- Middle-click closes tab. Ctrl+W closes active tab.

**Tab content**: Each tab renders `<FileViewer file={tab.file} currentPath={tab.currentPath} />`.

### Feature 6: WebSocket Upload Notifications

**Server** (`server/src/ws.ts`):
- Use `@fastify/websocket` plugin (added to Fastify instance in app.ts).
- `GET /api/ws` — WebSocket upgrade endpoint.
- On connection: add client to a `Set<WebSocket>`.
- On close/error: remove from set.
- Heartbeat: `setInterval` every 30s sends ping frame.
- Export `broadcast(event: object)` function.
- In `uploadRoutes`, after successful upload: `broadcast({ type: 'upload', path: targetPath, files: uploaded.map(f => f.name) })`.

**Frontend** (`client/src/hooks/useWebSocket.ts`):
```typescript
function useWebSocket(onUpload: (event: UploadEvent) => void): WebSocket | null
```
- Connects to `ws://${location.host}/api/ws` on mount.
- On `message`: parse JSON. If `type === 'upload'`, call `onUpload(event)`.
- Auto-reconnect with exponential backoff (1s, 2s, 4s, max 30s).
- Cleanup on unmount.

**Integration in App.tsx**: `useWebSocket` calls `handleRefresh()` to reload the current file list after an upload.

### Feature 7: Tree View File List

**Component**: `FileTree` (replaces `FileBrowser`)

```
Props: {
  selectedFilePath: string | null;  // path of selected file
  openTabs: Tab[];                  // for highlighting open tab files
  onSelectFile: (item: FileItem, path: string) => void;
  onNavigate: (path: string) => void;
  onRefresh: () => void;
}
```

**Data structure**:
```typescript
interface TreeNode {
  item: FileItem;
  path: string;       // full relative path
  depth: number;
  expanded: boolean;
  children?: TreeNode[];
  loaded: boolean;    // whether children have been fetched
}
```

**Loading strategy**: Lazy. On mount, fetch root (`path=''`). Only directories are expandable. Clicking expand on a directory: if `!loaded`, fetch `fetchFileList(dirPath)` and populate `children`. Toggle `expanded`.

**Rendering**:
- Flat list of visible nodes (expanded directories show their children inline with indentation).
- Each row: `[chevron / spacer] [icon] [name] [size]`.
- Directories always sorted first, then files, both alphabetically.
- Hidden files (dotfiles) excluded (consistent with server filtering).
- Selected file row has accent background.
- Single-click on file: opens/activates tab. No double-click needed.

**Keyboard**: ArrowUp/Down to navigate, ArrowRight to expand folder, ArrowLeft to collapse folder, Enter to open.

**Props difference from FileBrowser**: No `path`/`onNavigate` for directory browsing — the tree handles all navigation internally. `onRefresh` still needed. `onSelectFile` replaces `onSelect`.

### Feature 8: Right-Click Context Menu

**Component**: `ContextMenu`

```
Props: {
  x: number;
  y: number;
  node: TreeNode;
  onClose: () => void;
  onNewFolder: (parentPath: string) => void;
  onRename: (itemPath: string, name: string) => void;
  onCopy: (itemPath: string, name: string) => void;
  onDelete: (itemPath: string, name: string) => void;
  onDownload: (itemPath: string) => void;
  onRefresh: () => void;
}
```

**Behavior**:
- `onContextMenu` on tree rows. Calls `e.preventDefault()`, positions menu at `{x: e.clientX, y: e.clientY}`.
- Menu rect: white background (light) or dark surface (dark), border, rounded corners, shadow/backdrop-blur.
- Menu items: **New Folder**, separator, **Rename / Move**, **Copy**, **Delete**, **Download** (files only), separator, **Refresh**.
- Each item has an icon and label. Hover highlights with accent background.
- Close on: click outside (`document.addEventListener('click', ...)`), Escape key, or clicking a menu item.
- Clicking a menu item invokes the corresponding action, which opens the existing Modal dialogs (renamed/adapted for the new component structure).

**The existing action buttons** (✎ ⎘ ✕) on each file row are **removed** — replaced entirely by the context menu.

## Data Flow

```
1. User clicks "Upload" → FileUploader.uploadFiles()
   → POST /api/upload/multi → server saves file
   → server broadcasts WS { type: 'upload', ... }
   → useWebSocket receives → calls handleRefresh()
   → FileTree reloads current expanded nodes

2. User clicks file in tree → App: openTab(file, path)
   → TabBar shows new/activated tab
   → PreviewPanel renders FileViewer for active tab

3. User right-clicks tree node → ContextMenu opens
   → User clicks "Rename" → Modal opens (same existing pattern)
   → On confirm → API call → onRefresh → tree reloads

4. User drags ResizeHandle → sidebar width CSS variable updated
   → all flex children reflow

5. User clicks ThemeToggle → localStorage updated
   → data-theme attribute toggled → CSS variables switch
   → all components re-render with new colors
```
